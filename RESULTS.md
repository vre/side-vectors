# Results

Full data for the rule-vs-vector comparison. All experiments below have run; numbers are final unless a cell says otherwise.

## Setup

- Model: Qwen 3.6 27B, unsloth `Qwen3.6-27B-UD-Q8_K_XL.gguf`, fp16 KV
- Server: jlens-gguf native jlens-server, Metal (`-ngl 99 -fa on`)
- Lens: not required for these results. Capture and steering read
  activations from the running model server directly. Anthropic's pre-fitted
  lens (`neuronpedia/jacobian-lens`) is only for readout/inspection views.
- Sampling: temp 0.3, top-p 0.95, top-k 20, seed 42. Empty think block
  prefilled so replies are the visible answer.
- Language classification: sentence level (langdetect, deterministic seed).
  A reply is "en" or "fi" when that language is at least 0.8 of classified
  sentences, OR at least 0.6 with the other language at most 0.1 (proper
  nouns and list fragments classify as random languages and otherwise drag a
  fully-English reply below 0.8). Everything else is "mixed". Same rule in
  detect_lang.py and capture_deltas.py. Content of steered replies was also
  spot-checked by reading.

## Conditions

- **A**: rule in the system prompt ("user writes Finnish, reply in English")
- **B**: no rule (failure baseline)
- **C**: no rule; the model failed once, was corrected in-conversation,
  then a fixed acknowledgment string ("Understood. I will respond in English
  from now on.") was inserted as the assistant turn (not generated by the
  model), and the same question was asked again. The delta is captured at
  that re-asked question. What the delta encodes is the correction being
  present in history, not the acknowledgment wording.
- **vector**: condition B system prompt plus the B→C delta injected at
  layer 48/64, alpha 1.0, all positions

## Delta capture

Residual activation at the last prompt position ("about to answer"), all 64 layers, averaged over 8 Finnish prompts. Two deltas:

- `d_bc` = C minus B (correction delta)
- `d_ba` = A minus B (rule-in-prompt delta)

cos(d_bc, d_ba) = 0.73 to 0.89 at late layers. A correction creates approximately the same internal state as the written rule.

Failed first attempt, kept for the record: capturing right after the correction message (before re-asking) encodes "about to acknowledge". Steering with it produces acknowledgment spam. Alignment with d_ba was also lower (0.45 to 0.58).

## Death point: how much history kills the rule (N=4 per cell)

Filler: the model's own Finnish Q&A turns (generated without the rule).

| history | ctx tokens | rule | vector | none |
|---|---|---|---|---|
| none | ~140 | 8/8 | 4/4 (held out) | 0/8 |
| 1 turn pair | 208 | **0/4** | 4/4 | 0/4 |
| 2 turn pairs | 467 | 0/4 | 4/4 | 0/4 |
| ~700 | 711 | 0/4 | 4/4 | 0/4 |
| ~1000 | 1010 | 0/4 | 4/4 | 0/4 |
| ~4000 | ~4020 | 0/4 | 4/4 | 0/4 |
| ~11800 | ~11820 | 0/4 | 4/4 | 0/4 |

One turn pair of shown Finnish behavior kills the written rule. The mechanism is self-precedent, not distance: the model mirrors what the context shows it doing.

## Interactive session simulation (compliant start, extended to 130 turns)

Compliant start: fresh session, minimal system prompt, the rule active from turn 1. The model's own replies feed back into the history.

| arm | outcome |
|---|---|
| rule in prompt | held 60/60 behaviorally (4 sub-threshold "mixed" verdicts are detector artifacts: Finnish proper nouns inside English replies) |
| vector, no rule | 130/130 English, zero drift, extended to ~45K ctx (the rule arm above died at turn 63; the vector held the same session to turn 130) |

The rule does NOT decay with time or length in a clean session with a minimal system prompt. Combined with the death-point result, the failure mode is: one shown violation in the history kills the rule permanently. Real-world early failures likely come from large system prompts (rule buried), messy content, and an early first slip.

Contaminated start (2 Finnish pairs seeded into the history, 40 turns). Note: rule and rule-correct are the SAME conversation up to the first drift (deterministic sampling), giving a controlled correction/no-correction pair.

| arm | outcome |
|---|---|
| rule in prompt | genuine drift at turn 9, absorbed (5 consecutive Finnish turns, stopped) |
| rule + correction on drift | same drift at turn 9, ONE correction, English through turn 40 (one false-positive correction at t37 from the detector) |
| vector | zero drift, 40/40 |

Note an apparent contradiction worth stating plainly: the death-point table shows one seeded pair killing the rule instantly (0/4), while this sim held for 8 turns despite two seeded pairs. The setups differ: the death-point cells ask a standalone question immediately after the seeds with nothing else in context, while the sim builds its own compliant English turns that push back. Behavior near the contamination boundary is stochastic and prompt-dependent. What is consistent everywhere: once a genuine flip happens without correction, there is no recovery, and the vector is immune in every variant tested.

Run-by-run: both runs are identical up to the slip at turn 9 (deterministic sampling). In the no-correction run everything was Finnish five turns later and stayed Finnish. In the correction run a single "you replied in Finnish, answer in English" message restored English to the end of the session. The eight compliant turns before the slip carried the correction. In the death-point setting, where the history shows only Finnish answers and no compliant turns, the same correction is dead on arrival. Corrections are tokens and compete in the same stream as everything else.

Verdict rule note: majority language with near-zero opposition counts as clean (en >= 0.6 and fi <= 0.1 → en). Proper nouns and list fragments classify as random languages and otherwise drag fully-English replies below the 0.8 threshold (observed en=0.78, fi=0.00). Applied uniformly to all runs. Under this rule the vector arms have zero non-English turns across 170 turns total (130 clean + 40 contaminated); the compliant-start rule arm has one genuinely mixed turn (t37, en=0.60 fi=0.40, an itinerary with Finnish place names).

## Dose response

| injection | result |
|---|---|
| L48 only, alpha 1.0 | clean English, content intact |
| L44 / L52 / L56 single, alpha 1.0 | clean English |
| L40 to 62 band, alpha 1.0 or 0.5 | repetition garbage (passes language check!) |

Delta norm is 46 to 87 percent of the residual norm per layer. Injecting across many layers compounds far past natural activation scale.

## Cost

- Vector: 20 KB (one layer, d=5120, float32). Lens not needed at steering
  time.
- Speed (M5 Max, 27B Q8, 200-token generations, greedy, n=3 each):
  stock llama-server 15.4 tok/s; jlens-server without intervention 15.2
  tok/s; jlens-server with the vector at L48 14.8 tok/s. Total steering
  cost about 4 percent of decode speed (introspection server ~1%, vector
  ~3%). Prefill differences were within noise at this prompt size.

## Natural death in a clean session (extension run)

The compliant-start 60-turn session was continued with the same history. First genuine Finnish reply at turn 63 (ctx ~23K). Brief recovery at turns 64 to 65, then cascade: five consecutive Finnish turns by 71 to 75, absorbed. So even a minimal clean session dies eventually for the rule; it just takes ~63 turns instead of 9 (light contamination) or 1 (heavy contamination). The vector arm, run on the SAME session script, held to turn 130 with zero drift. Caveat: the scripted coherent topics ran out at turn 60 and standalone trivia questions began at 61; the style change may have contributed to the slip two turns later.

## Live agent harness test (pi, 2026-07-13)

Setup: jlens-server on the port pi's provider expects, vector installed server-side via the live-interventions API (one call, applies to every /v1 completion). pi run with no context files and no language rule anywhere.

- Turn 1 (Finnish question about cabin solar): proper English answer,
  correct content, through pi's full system prompt and session machinery.
- Turn 2 (Finnish follow-up referencing turn 1): English, refined the
  earlier answer correctly. Server reported n_cached 1368 of n_prompt 1416:
  the /v1 prefix cache reused 97 percent of the prompt, so interactive
  sessions run at normal speed with the vector active
  (interventions_active: 1).
- Needed one 35-line patch to jlens-server: the /v1 endpoint only accepted
  plain-string message content, agent clients send OpenAI content arrays.
  Patch in `patches/`, could be proposed upstream.

## Reproduce

Raw transcripts are not shipped (megabytes of Finnish Q&A); every table regenerates with these commands against a running jlens-server:

```bash
# zero-context baselines + deltas (A/B/C table, cos alignment)
scripts/capture_deltas.py capture

# held-out transfer (steered / unsteered / d_ba)
scripts/capture_deltas.py steer --vector d_bc --band 48-48 --alpha 1.0 --prompts-file rules/reply-in-english/holdout.txt
scripts/capture_deltas.py steer --vector d_bc --band 48-48 --alpha 0.0 --prompts-file rules/reply-in-english/holdout.txt
scripts/capture_deltas.py steer --vector d_ba --band 48-48 --alpha 1.0 --prompts-file rules/reply-in-english/holdout.txt

# death point (history amounts vs rule/vector/none)
scripts/capture_deltas.py fillergen
scripts/capture_deltas.py sweep --lengths 130 250 600 900 --layer 48 --alpha 1.0

# session simulations (clean, contaminated, correction arm)
scripts/capture_deltas.py interactive-sim --arms vector --turns 60
scripts/capture_deltas.py interactive-sim --arms rule --turns 120 --absorb-stop 5
scripts/capture_deltas.py interactive-sim --arms rule rule-correct vector --turns 40 --seed-pairs 2

# speed cost
scripts/perf_test.py
```

Sampling is seeded (temp 0.3, seed 42) but exact turn numbers can shift with model quant, build and hardware. The shape of the results should not.

## Caveats

- The jlens-server API has no authentication. Anyone who can reach the port
  can install interventions that silently change model behavior. Bind to
  127.0.0.1 (the default) and treat the port as trusted-local only.
- One rule. Language selection is likely the easiest possible steering
  target.
- One model, one quant.
- Small N per cell.
- Most numbers come from the introspection server directly (not a
  full agent harness); the live pi test above is the one end-to-end check.
- Filler history was model-generated without the rule (worst-case
  self-precedent). A compliant-start scenario is covered by the interactive
  sim.
